## 4. Тестові запити та 5. Приклади відповідей агента

### Тестовий запит №1: Детекція короткого замикання на магістральній ЛЕП

> **Вхідні дані користувача:**
> * **Сценарій:** Раптове трифазне коротке замикання (КЗ) на лінії 110 кВ.
> * **Метод:** Глибокий щільний Autoencoder (Dense Autoencoder).
> * **Специфіка:** Потрібна мінімальна затримка (Latency) для релейного захисту.
> 
> 

#### Приклад відповіді агента №1:

**1. Назва аварійного сценарію:**
Детекція трифазного короткого замикання (КЗ) на магістральній ЛЕП 110 кВ.

**2. Джерела даних:**

* *Джерело:* Високочастотні лог-файли з терміналів РЗА (релейного захисту) та датчиків PMU (Phasor Measurement Units).
* *Ознаки (Features):* Трьохфазні напруги ($U_A, U_B, U_C$), трьохфазні струми ($I_A, I_B, I_C$), частота мережі ($f$), фазові кути.

**3. Pipeline обробки даних:**

* **Збір:** Потік даних з частотою 1 кГц.
* **Вікно:** Ковзне вікно (Sliding window) розміром 50 мс (50 відліків).
* **Нормалізація:** MinMaxScaler (приведення до діапазону $[0, 1]$), розрахований на режимі нормального навантаження.

$$X_{norm} = \frac{X - X_{min}}{X_{max} - X_{min}}$$



**4. Архітектура моделі Autoencoder:**

* *Input Layer:* 600 нейронів (6 ознак $\times$ 50 відліків вікна).
* *Encoder Layers:* Dense(256, ReLU) $\rightarrow$ Dense(64, ReLU) $\rightarrow$ Dense(16, ReLU) [Latent Space].
* *Decoder Layers:* Dense(64, ReLU) $\rightarrow$ Dense(256, ReLU) $\rightarrow$ Dense(600, Sigmoid) [Output].

**5. Агентна логіка (Decision-making):**
Агент обчислює помилку реконструкції (MSE) для кожного вікна.


$$\text{MSE} = \frac{1}{n}\sum_{i=1}^{n}(X_{true} - X_{pred})^2$$


*Policy:* Якщо $\text{MSE} > \text{Threshold}$ протягом 3-х тактів поспіль $\rightarrow$ Агент приймає рішення: **«АВАРІЯ: Коротке замикання»** і видає сигнал на вимкнення вимикача ЛЕП.

**6. Інтерфейс:**

* *API:* RESTful API на FastAPI. Метод `/api/v1/predict` приймає JSON-масив параметрів мережі, повертає статус (`Normal` / `Anomaly`) та значення MSE.

**7. Метрики якості:**

* **F1-Score:** $> 0.98$ (критично для уникнення хибних спрацювань).
* **RMSE:** $< 0.02$ на нормальних даних.
* **Latency:** $< 5\text{ мс}$ (час інференсу моделі).

**8. Приклад коду на Python:**

```python
import numpy as np
from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import Dense

# Симуляція нормальних даних (ток/напруга)
X_train = np.random.rand(1000, 600) 

# Побудова Autoencoder
model = Sequential([
    # Encoder
    Dense(256, activation='relu', input_shape=(600,)),
    Dense(16, activation='relu'), 
    # Decoder
    Dense(256, activation='relu'),
    Dense(600, activation='sigmoid')
])

model.compile(optimizer='adam', loss='mse')
model.fit(X_train, X_train, epochs=5, batch_size=32, verbose=0)

# Визначення порогу (Threshold)
reconstructions = model.predict(X_train)
mse = np.mean(np.power(X_train - reconstructions, 2), axis=1)
threshold = np.percentile(mse, 99) # 99-й перцентиль
print(# Поріг аномалії: {threshold}")

```

**9. Очікуваний результат виконання:**
Під час замикання струм різко зростає, а напруга падає. Модель Autoencoder, яка тренувалася лише на нормальних синусоїдах, не зможе відновити цей сигнал. Помилка реконструкції (MSE) підскочить у 50-100 разів вище за поріг, агент миттєво зафіксує аномалію.

**10. Критерії оцінювання:**

* Швидкість реакції (Latency) не повинна перевищувати 10 мс.
* Відсутність False Positive (хибних спрацювань) під час планових перемикань навантаження.

### Тестовий запит №2: Виявлення перевантаження трансформаторних підстанцій

> **Вхідні дані користувача:**
> * **Сценарій:** Поступове термічне перевантаження трансформатора 10/0.4 кВ через нелегальні підключення.
> * **Метод:** LSTM Autoencoder (для врахування часових трендів).
> * **Специфіка:** Дані з лічильників Smart Grid (АСКОЕ).
> 
> 

#### Приклад відповіді агента №2:

**1. Назва аварійного сценарію:**
Детекція хронічного теплового та потужнісного перевантаження силового трансформатора.

**2. Джерела даних:**

* *Джерело:* Погодинні дані з інтелектуальних систем обліку електроенергії (Smart Meters).
* *Ознаки:* Активна потужність ($P$), реактивна потужність ($Q$), температура верхніх шарів мастила ($T_{oil}$), температура довкілля.

**3. Pipeline обробки даних:**

* **Збір:** Агрегація даних з дискретністю 30 хвилин.
* **Заповнення пропусків:** Лінійна інтерполяція для відновлення втрачених пакетів від лічильників.
* **Стандалізація:** StandardScaler (масштабування до середнього 0 та дисперсії 1).

**4. Архітектура моделі Autoencoder:**

* *Input:* Тензор розмірності `(Batch, Timesteps=48, Features=4)` — історія за останні 24 години.
* *Encoder:* LSTM(32, return_sequences=False).
* *Repeat Vector:* Повторення прихованого стану 48 разів.
* *Decoder:* LSTM(32, return_sequences=True) $\rightarrow$ TimeDistributed(Dense(4)).

**5. Агентна логіка (Decision-making):**
*Policy:* Агент порівнює реальний графік нагріву й споживання з відновленим «нормальним». Якщо MAE (Mean Absolute Error) по температурі перевищує ліміт протягом 5 годин поспіль, агент надсилає диспетчеру попередження: *«Увага: Ризик пошкодження ізоляції через перевантаження»*.

**6. Інтерфейс:**

* *UI:* Веб-панель (Dashboard) на Grafana / Streamlit, де підсвічуються трансформатори з червоним рівнем аномальності (Anomaly Score).

**7. Метрики якості:**

* **Precision:** $> 0.92$
* **Recall:** $> 0.95$ (важливо не пропустити деградацію трансформатора).
* **RMSE:** за вихідними ознаками.

**8. Приклад коду на Python:**

```python
import tensorflow as tf
from tensorflow.keras.layers import LSTM, RepeatVector, TimeDistributed, Dense

# Модель LSTM Autoencoder для часових рядів
model = tf.keras.Sequential([
    LSTM(32, activation='relu', input_shape=(48, 4), return_sequences=False),
    RepeatVector(48),
    LSTM(32, activation='relu', return_sequences=True),
    TimeDistributed(Dense(4))
])
model.compile(optimizer='adam', loss='mae')
print("Модель для часових рядів успішно створена.")

```

**9. Очікуваний результат виконання:**
Система виявляє аномальний тренд зростання температури мастила при незмінній температурі повітря, сигналізуючи про внутрішні дефекти або крадіжку електроенергії.

**10. Критерії оцінювання:**
Виявлення дефекту мінімум за 12 годин до досягнення критичної (аварійної) температури відключення.

### Тестовий запит №3: Феномен ферорезонансу в мережі (Високочастотна аномалія)

> **Вхідні дані користувача:**
> * **Сценарій:** Виникнення ферорезонансних процесів при перемиканнях холостого ходу.
> * **Метод:** Convolutional (CNN) Autoencoder для аналізу форми хвилі.
> 
> 

#### Приклад відповіді агента №3:

**1. Назва аварійного сценарію:**
Реєстрація та класифікація ферорезонансних явищ в трансформаторах напруги.

**2. Джерела даних:**

* *Джерело:* Цифрові реєстратори аварійних подій (ЦРАП / осцилографи).
* *Ознаки:* Необроблені миттєві значення напруги фаз (Waveform data, частота 5 кГц).

**3. Pipeline обробки даних:**

* Сегментація осцилограм на фрагменти по 200 мс.
* Застосування Швидкого Перетворення Фур'є (FFT) для отримання спектральних ознак (додатково).
* Нормалізація сигналу в діапазон $[-1, 1]$.

**4. Архітектура моделі Autoencoder:**

* *Input:* 1D-сигнал розміром `(1000, 1)` (0.2 секунди запису).
* *Encoder:* Conv1D(16, kernel_size=3) $\rightarrow$ MaxPool1D $\rightarrow$ Conv1D(8, kernel_size=3) $\rightarrow$ MaxPool1D.
* *Decoder:* UpSampling1D $\rightarrow$ Conv1D_Transpose(16) $\rightarrow$ Conv1D_Transpose(1).

**5. Агентна логіка (Decision-making):**
*Policy:* Ферорезонанс сильно спотворює синусоїду (виникають субгармоніки). CNN-автокодувальник не зможе згенерувати таку форму хвилі. При перевищенні ліміту помилки реконструкції на 300% агент активує тригер запису аварійного дампу даних.

**6. Інтерфейс:**

* *CLI:* Консольний скрипт для фонової перевірки та парсингу файлів формату COMTRADE.

**7. Метрики якості:**

* **Accuracy:** $> 0.96$
* **F1-Score:** $> 0.94$
* **Latency:** $< 50\text{ мс}$ (допустимо для даного типу аналізу).

**8. Приклад коду на Python:**

```python
from tensorflow.keras.layers import Conv1D, MaxPooling1D, UpSampling1D, Conv1DTranspose
from tensorflow.keras.models import Sequential

model = Sequential([
    # Encoder
    Conv1D(16, 3, activation='relu', padding='same', input_shape=(1000, 1)),
    MaxPooling1D(2, padding='same'),
    # Decoder
    Conv1DTranspose(16, 3, activation='relu', padding='same'),
    UpSampling1D(2),
    Conv1DTranspose(1, 3, activation='linear', padding='same')
])
model.compile(optimizer='rmsprop', loss='mse')
print("СNN-Autoencoder для аналізу форми хвилі скомпільовано.")

```

**9. Очікуваний результат виконання:**
Ферорезонансна крива чітко детектується як аномалія, що дозволяє зберегти дороге обладнання від теплового вибуху.

**10. Критерії оцінювання:**
Правильна фільтрація нормальних комутаційних сплесків від небезпечного сталого ферорезонансу.

## 6. Аналіз переваг і недоліків розробленого агента

По завершенню тестування агента, студент формує аналітичний висновок:

### Переваги (Pros):

* **Навчання без вчителя (Unsupervised):** Нам не потрібні розмічені історичні дані аварій (яких в реальності дуже мало). Модель вчиться виключно на стабільній, нормальній роботі енергосистеми.
* **Універсальність архітектури:** Залежно від вхідних джерел (PMU, Smart Meters, COMTRADE), агент адаптує структуру (Dense, LSTM чи CNN Autoencoder).
* **Швидкість (Latency):** Час інференсу нейромережі складає лічені мілісекунди, що задовольняє вимоги релейного захисту.

### Недоліки (Cons):

* **Проблема вибору порогу (Threshold):** Якщо встановити надто низький поріг, система буде генерувати хибні тривоги (False Positive) при різких, але безпечних змінах навантаження (наприклад, запуск заводу).
* **Чутливість до дрейфу концепту (Concept Drift):** Зміна сезонів (зима/літо) змінює базовий профіль споживання енергії. Агенту необхідне періодичне перенавчання (Retraining), інакше нормальний літній режим буде сприйматися як аномалія.
* **Чорна скринька (Black Box):** Autoencoder каже *що* є аномалія, але сам по собі не може пояснити *чому саме* вона сталася (потрібні додаткові інструменти на кшталт SHAP/LIME).
