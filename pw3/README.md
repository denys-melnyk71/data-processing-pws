## 1. ВИБІР БІБЛІОТЕК

* **`pandas`:** Головний інструмент для роботи зі структурованими даними у Python. Методом `pd.read_csv()` імпортуємо дані, а об'єкт `DataFrame` надає можливість приведення через `.astype()`. Метод `.memory_usage(deep=True)`, а саме його прапорець `deep=True` змушує Pandas аналізувати реальні системні витрати пам'яті на зберігання об'єктів у RAM (зокрема, залишкові текстові посилання), а не просто оцінювати структуру «на папері».
* **`gc` (Garbage Collector):** Вбудований модуль Python для керування пам'яттю. Застосування `gc.collect()` після операцій видалення колонкм `Datetime` дозволяє визначити різницю у використанні RAM до та після оптимізації.

## 2. МАТЕМАТИЧНЕ ТА АРХІТЕКТУРНЕ ПІДҐРУНТЯ

В основі оптимізації пам'яті лежить перерозподіл бітової сітки в архітектурі комп'ютера. За замовчуванням Pandas виділяє максимальну розрядність: 64 біти (8 байтів) на кожне число.

### Математичні формули місткості типів даних:

Для знакових цілих чисел (`int`) діапазон значень при розрядності $b$ бітів обчислюється як:


$$[-2^{b-1}; 2^{b-1} - 1]$$

Для беззнакових цілих чисел (`uint`), які ми застосували для календарних компонентів, від'ємна частина відсутня, тому місткість розширюється в додатну сторону:


$$[0; 2^b - 1]$$

### Перерозподіл пам'яті в програмі:

1. **Календарні метрики (Місяць, День, Година):** Значення не перевищують 31. Дефолтний `int64` (8 байтів) замінено на `uint8` ($b=8$).

$$\text{Діапазон } uint8: [0; 2^8 - 1] = [0; 255]$$



*Економічний ефект:* Зменшення витрат пам'яті на один рядок з 8 байтів до 1 байта (**у 8 разів**).
2. **Рік:** Значення у датасеті чотиризначні (наприклад, 2002). В `uint8` вони не влазять, тому застосовано `uint16` ($b=16$).

$$\text{Діапазон } uint16: [0; 2^{16} - 1] = [0; 65535]$$

*Економічний ефект:* Зменшення витрат з 8 байтів до 2 байтів (**у 4 рази**).
3. **Енергоспоживання (`PJME_MW`):** Показники мережі коливаються в межах п'ятизначних чисел (~20 000 – 35 000). За замовчуванням Pandas зчитує дробові числа як `float64` (8 байтів). Конвертація у `float32` виділяє під мантису та експоненту 32 біти (4 байти), чого цілком достатньо для збереження інженерної точності вимірів.
*Економічний ефект:* Зменшення витрат з 8 байтів до 4 байтів (**удвічі**).

### Ефективність стиснення:

Обчислюється як відсоткове відношення заощадженої пам'яті до початкового стану:

$$\eta = \frac{M_{\text{before}} - M_{\text{after}}}{M_{\text{before}}} \times 100\%$$

## 3. ПІДХІД ТА МЕТОДИ АНАЛІЗУ ДАНИХ

У роботі застосовано комбінацію двох підходів:

1. **Розбиття дати:** Оскільки оригінальний формат дати є складним для лінійних моделей, вона розбивається на 4 окремі компоненти (`Year`, `Month`, `Day`, `Hour`).
2. **Конвертація типів даних** Після витягування інформації, оригінальна колонка `Datetime` стає надлишковою (дублюючою). Її видалення з одночасним урізанням розрядності числових типів через `.astype()` мінімізує системний оверхед.

### Вид аналізу:

Програма не аналізує саму суть енергетичного колапсу чи споживання, а досліджує метадані: фізичну вагу структури даних у RAM комп'ютера, оптимізуючи її під потреби Big Data систем.
